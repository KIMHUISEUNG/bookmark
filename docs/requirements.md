# 요구사항 명세서

# 북마크/리딩 리스트 요구사항 명세서

## 0. 프로젝트 한 줄

- 웹에서 발견한 콘텐츠(URL)를 **저장·태그·검색·정렬**하고, **읽음 상태/요약 메모**로 관리하는 리딩 리스트

---

## 1. 문제/목표

### 1.1 문제

- 링크를 저장해도 나중에 못 찾는다(분류/검색/정리가 약함).
- 읽을거리의 상태(읽을 예정/읽는 중/완료/보류) 관리가 안 된다.
- 저장한 링크의 핵심을 잊는다(메모/하이라이트 부족).

### 1.2 목표

- 링크를 빠르게 저장하고(1분 내), 태그/검색으로 5초 내 찾게 한다.
- 읽음 상태로 “이번 주 읽을 것”을 관리할 수 있게 한다.
- 각 링크에 짧은 메모를 남길 수 있게 한다.

### 1.3 추후 목표

- 크롬 확장프로그램
- 본문 전체 크롤링/오프라인 저장
- 소셜 기능(팔로우/피드 등)

---

## 2. 페르소나(1명)

- 이름/상황: “희성”, 취준 중, 기술 글/강의 링크를 매일 저장함
- 목표: 나중에 면접/공부 때 바로 찾고, 읽은 것/안 읽은 것을 구분하고 싶음
- 페인 포인트: 북마크가 쌓이면 검색이 안 되고, 태그가 없으면 다시 못 찾음
- 사용 환경: 데스크톱(크롬) 중심 + 가끔 모바일

---

## 3. 핵심 사용자 여정

1. 저장
   - URL 입력 → 자동 제목/썸네일(가능하면) → 태그 선택/추가 → 저장
2. 탐색/정리
   - 리스트에서 검색/필터(태그/상태) → 정렬(최근 저장/제목/읽음 날짜) → 상태 변경
3. 읽기 기록
   - 상세 페이지에서 메모 작성 → 읽음 완료 처리 → “완료됨” 목록에서 회고

---

## 4. MVP 범위

### 4.1 Must

- 링크 저장(제목, URL, 태그, 상태)
- 리스트 보기(검색/필터/정렬 최소 1개 이상)
- 상세 보기(메모, 상태 변경)
- CRUD(수정/삭제)
- 기본 오류 처리(빈 URL, 중복 URL, 네트워크/저장 실패)

### 4.2 Should

- 태그 자동완성/관리(태그 목록, 태그 이름 변경)
- 읽을 예정/읽는 중/완료/보류 상태 + 카운트
- URL 메타데이터(제목/설명/썸네일) 자동 가져오기
- 빈 상태 UI, 로딩 스켈레톤

### 4.3 Could (고도화)

- 컬렉션/폴더
- 공유 링크(읽기 전용 공개)
- 즐겨찾기/핀, 아카이브
- 가져오기/내보내기(CSV/JSON)
- 단축키(저장/검색 포커스)

## 5. 기능 요구사항

### US-01 링크 저장

- User Story: 사용자는 URL을 저장해서 나중에 다시 접근하고 싶다.
- Acceptance Criteria
  - [ ] URL 입력 없이 저장 시, 필수 입력 오류가 표시된다.
  - [ ] 유효하지 않은 URL 형식이면 저장되지 않고 안내한다.
  - [ ] 저장 성공 시 리스트에 즉시 반영된다.

### US-02 저장 시 기본 정보 입력

- User Story: 사용자는 제목/태그/상태를 같이 저장하고 싶다.
- Acceptance Criteria
  - [ ] 제목은 사용자가 수정 가능하다(자동 값이 있어도).
  - [ ] 태그는 0개 이상 가능하며 중복 태그는 허용하지 않는다.
  - [ ] 상태 기본값은 “읽을 예정”이다.

### US-03 리스트 탐색(검색)

- User Story: 사용자는 키워드로 저장된 링크를 빠르게 찾고 싶다.
- Acceptance Criteria
  - [ ] 검색어가 제목/URL/메모(선택)에서 매칭된다.
  - [ ] 검색 결과가 0개면 빈 결과 메시지를 보여준다.
  - [ ] 검색어를 지우면 전체 목록으로 돌아온다.

### US-04 필터(태그/상태)

- User Story: 사용자는 태그/상태로 목록을 좁히고 싶다.
- Acceptance Criteria
  - [ ] 태그 필터를 선택하면 해당 태그가 포함된 항목만 보인다.
  - [ ] 상태 필터 선택 시 해당 상태만 보인다.
  - [ ] 필터를 해제하면 원래 목록으로 복구된다.

### US-05 정렬

- User Story: 사용자는 최근 저장한 순으로 보고 싶다.
- Acceptance Criteria
  - [ ] 기본 정렬은 “최근 저장순(내림차순)”이다.
  - [ ] 정렬 옵션 최소 2개(최근 저장/제목 오름차순 등)를 제공한다.

### US-06 상세 보기

- User Story: 사용자는 링크의 자세한 정보를 보고 관리하고 싶다.
- Acceptance Criteria
  - [ ] 상세에서 제목/URL/태그/상태/메모를 확인할 수 있다.
  - [ ] URL 클릭 시 새 탭으로 열린다.

### US-07 메모

- User Story: 사용자는 링크에 짧은 요약/메모를 남기고 싶다.
- Acceptance Criteria
  - [ ] 메모는 저장/수정/삭제 가능하다.
  - [ ] 메모 저장 중 로딩 상태가 표시된다(또는 버튼 비활성화).

### US-08 상태 변경

- User Story: 사용자는 읽음 상태를 변경하고 싶다.
- Acceptance Criteria
  - [ ] 리스트 또는 상세에서 상태 변경이 가능하다.
  - [ ] “완료”로 변경 시 완료 일시가 기록된다(선택 구현).

### US-09 수정/삭제

- User Story: 사용자는 저장한 링크를 수정하거나 삭제하고 싶다.
- Acceptance Criteria
  - [ ] 수정 시 값이 즉시 반영된다.
  - [ ] 삭제는 확인(모달/토스트) 후 수행된다.
  - [ ] 삭제 후 되돌리기(Undo)는 Could(선택).

### US-10 중복 처리

- User Story: 사용자는 같은 URL을 중복 저장하지 않게 하고 싶다.
- Acceptance Criteria
  - [ ] 동일 URL 저장 시 “이미 저장됨” 안내 + 기존 항목으로 이동(선택) 또는 업데이트 옵션 제공.

## 6. 비기능 요구사항

- 접근성: 키보드 탐색 가능(탭 이동/버튼 포커스), 폼 라벨/에러 메시지 제공
- 성능: 리스트 200개 기준 스크롤/검색이 버벅이지 않음(필요 시 디바운스)
- 오류 처리: 저장/불러오기 실패 시 사용자 메시지 + 재시도 가능
- 보안/권한: 로그인 기능이 없으면 개인 데이터는 로컬 저장으로 제한(또는 공개 범위 없음)
- 테스트: 핵심 유스케이스 2~3개(저장/검색/필터) 단위 또는 통합 테스트
- 배포: Vercel 등으로 배포, README에 설치/기능/트레이드오프 기록

## 7. 데이터/화면 목록(간단히)

### 7.1 Bookmark

- id: string
- url: string (unique)
- title: string
- description?: string
- thumbnailUrl?: string
- status: `TO_READ | READING | DONE | PAUSED`
- tags: `Tag[]` (또는 `string[]`)
- note?: string
- createdAt: datetime
- updatedAt: datetime
- completedAt?: datetime

### 7.2 Tag

- id: string
- name: string (unique)
- createdAt: datetime

### 7.3 화면/라우팅(Next 기준)

- `/` : 전체 리스트(검색/필터/정렬/추가 버튼)
- `/new` : 새 북마크 추가(모달로 대체 가능)
- `/b/[id]` : 상세(메모, 상태 변경, 수정/삭제)
- `/tags` : 태그 관리(Should)
- `/settings` : 데이터 내보내기/가져오기(성장 단계)
